  • add domain user John

Gary S. Weaver added a comment - 04/Jan/12 11:52 AM - edited

If the issue were just that the user is listed but is not longer there/no longer a member, the workaround would be to possibly wait some period of time after user changes are made before the proper info shows up in Confluence. Then, tell the user to click on the refresh link in the CSUM page (not in the browser) to clear cache. At least that was the case in previous versions of Confluence.

In 2007-2008, there were issues with not using cache (iirc they were both performance related with large groups and possibly related to paging). That need may no longer be there, but I can't say for sure, and wouldn't want to go through the trouble to remove caching only to find out that it is still required. So, if the plugin caching needs to be removed, you'd need to get a Java developer to work on a custom build for it that you could test, and if that works, they would be welcome to commit those changes back into the project, as long as there was nothing environment specific in their changes.

However, in this case, It could be possible that the Confluence API/embedded Crowd API that the plugin is using is returning data that it shouldn't, indicating that there is a user membership for a user that doesn't exist.

If you would please attach debug logging (and please strip all customer specific/private data from it via text replacement) for the plugin, that might help a lot. We might need to add additional logging capability to get at the root of the problem- I'm not sure; I need to see the logs first to find out.

1111111;"sAMAccountName";"com.atlassian.user.impl.ldap.DefaultLDAPUser";"";

when user is joined any group "external members" table is updated with userid and group id
1111111;36306945
1111111;53444610
1111111;96075781

table "groups" contains group name and id relations

when user is removed from active directory, confluence can not find him anymore, BUT
SUSR plugin still shows information from group table - with existing and non existing users listed.

Trying to remove user from list, following link is used
https://site.address.com/customspacemgmt/permissions/docustompermissionsmanage.action?key=SPACEKEY&selectedGroup=spacekey-grouname&adminAction=removeUsersFromGroups&groups=spacekey-grouname&users=$generalUtil.urlEncode($user.name)&redirect=

as no user name is found in directory it fails.

Gary S. Weaver added a comment - 09/Jan/12 6:52 AM

Janis,

The problem is that I get the membership list from Atlassian's API, but we also use their API for paging. So, if they give me back null (empty) users in a group membership list, those null users are in the list of N users returned from the pager. Right now I'm just checking to see if the user objects are null in that list and hiding them if they are.

This is almost surely a bug in the Atlassian API. However, at the moment I don't have time to debug it, which would probably require writing a plugin that is as simple as possible but logs the membership list that comes back from the API and/or Pager to show that some users are null in the results. It would really help if you or another Java developer at your place of work could do this and follow-up with Atlassian. I'd love to help, but I support this plugin in my spare time and I've spent so much time assisting recently with Atlassian development that it might be a while before I can help much again. In addition, Atlassian is likely not going to support that version of Confluence fully or as quickly, so it would be great if you could try reproducing the issue in a copy of your current environment using the latest version of Confluence.

Good luck!
Gary
