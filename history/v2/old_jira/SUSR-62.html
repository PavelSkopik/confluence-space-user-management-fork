Description

There seems to be a bug when trying to configure the plugin to Autoprovision users Using LDAP Info.
After filling in the appropriate LDAP info including a test user ID and clicking on the Update button, the code generates a filter expression "null=<the_user_id>". This happens even if no value is supplied to the LDAP Narrowing Filter Expression textbox.

What seems to be happening is as follows:
1. LDAPHelper invokes getLDAPUser routine which invokes LDAPLookupUtil.getUserDetails routine with the ff code:
if (fContext != null)
{
String searchFilter;
if (fSearchFilterBase==null)

{ searchFilter = "(" + fEnv.get(LDAPUTIL_LDAP_ID_ATTRIBUTE) + "=" + userId.trim() + ")"; }

else

{ searchFilter = "(&" + fSearchFilterBase + "(" + fEnv.get(LDAPUTIL_LDAP_ID_ATTRIBUTE) + "=" + userId.trim() + "))"; }

LOG.info("Searching for user ["+userId+"] based on filter: " + searchFilter );
NamingEnumeration matchingEnum = null;
String searchBase=(String)fEnv.get(LDAPLookupUtil.LDAPUTIL_LDAP_SEARCH_BASE);
try
{
matchingEnum = fContext.search(searchBase, searchFilter, null, fControl);
2. the key line of code is fEnv.get(LDAPUTIL_LDAP_ID_ATTRIBUTE) .
fEnv object is a hashtable set by a previously-called subroutine LDAPLookupUtil.processOSuserConfig... here's the line
fEnv = osuserParser.getProviderPropertiesAsHash(requiredClass);
3. The osuserParser's getProviderPropertiesAsHash basically sets the hashtable key-value pairs from the osuser.xml; our log shows this line for the userID attribute:
2007-09-26 12:38:51,918 DEBUG WebContainer : 0 net.ldap.osuser.OSUserParser getProviderPropertiesAsHash Adding ldap property: uidAttribute = sAMAccountName
4. However, the key being retrieved in fEnv.get(LDAPUTIL_LDAP_ID_ATTRIBUTE) is defined as ff:
public static final String LDAPUTIL_LDAP_ID_ATTRIBUTE = "ldaputil.ldap.userIdLdapAttribute";
As a result, NULL is returned.
5. The atlassian application log confirms this:
2007-09-26 12:38:52,376 INFO WebContainer : 0 confluence.net.ldap.LDAPLookupUtil getUserDetails Searching for user a397280 based on filter: (null=a397280)
