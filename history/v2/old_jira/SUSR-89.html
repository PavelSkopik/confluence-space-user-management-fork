Some ideas:

1) anonymous sounds like it is not a space admin. The user must be a space admin to use CSUM. I don't think that it should spit out that error, but in earlier versions of Confluence non-space-managers could not see the space admin area, so if that is a new ability, please let me know. If it is something you've customized, then I don't know what we should do to help fix it in a generic way. I'm probably way off-base with that one though, right?

2) It really looks like it can't find the theme it is configured to use, and that is what is causing the error. My guess is that you replicated the DB and setup a new version of confluence and pointed it at that DB, but you didn't install all of the plugins/set it up yet. Am I way off base with that also? I would bet I'm 0 for 2 right now.

3) The breadcrumbs thing looks like the issue I pointed out in the ticket and in the comments that affects space admin in general (not just CSUM). Can you get into space admin? Even if you can, could you take a look at this page and see if it makes any sense?: http://confluence.atlassian.com/display/KB/CSP-18160+-+Error+when+accessing+Space%27s+Admin+console
I have a feeling that the issue you are having may be related to that issue. If it requires us to change the CSUM plugin, I'd be very surprised, but I'd do it. My guess is that Atlassian inadvertantly (in an attempt to get through this support issue quickly) did not fully assist you at getting to the root of this issue, which I think is likely a combination of themebuilder + theme you are using + changes in confluence somewhere around 2.8.

Best of luck and as always (or has been thus far since CSUM inception), we are here to help.

Gary
