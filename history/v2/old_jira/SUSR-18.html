Description

This is bit interesting:

I've enabled System.out.println in LDAPHelper.getLDAPUser(config, userid) method. which kind of helped to put these debug messages on console rather than in standard log file to stand out for quick verification.

And whenever I was trying to add users (new or existing users) or remove users from user group, I found that application makes calls LDAPHelper.getLDAPUser(config, remoteUser) remoteUser -> loggedin user.

Hence attribute messages, LDAP user information about those users get printed on console.

Not sure if this is a feature or bug. Thought of worth mentioning it here.

regards,
Rajendra
