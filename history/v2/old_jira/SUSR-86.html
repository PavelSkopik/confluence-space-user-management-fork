Gary S. Weaver added a comment - 31/Oct/08 8:50 AM

Aditee,

First try going into Confluence's own Administration area. If you can view group memberships from there without error (click on a group to see list of users), and you can do these repeatedly and can add/remove groups and add/remove users to those groups, then you should be able to use the CSUM plugin.

If you can't, then please contact Atlassian support and let them help you with that issue.

If you can view membership lists of groups, add/remove groups and users within Confluence Administration itself, then please turn on debug logging of the plugin as specified on the CSUM wiki page, and attach the debug logs to this ticket. But please don't do that until you have tried the other thing, because it is unlikely that we will be able to help you if this is a confluence configuration or other confluence issue.

Thanks!
Gary

Gary S. Weaver added a comment - 20/Aug/09 9:02 AM

Aditee,

Sorry that I didn't get back with you on this. Unfortunately, Cisco is the only place I know of personally that is using CSUM with Jira user/group management. I had thought that Rajendra Kadam, also of Cisco, said that you all were transitioning off of this and using something else instead? Perhaps that could resolve the issue. I was aware that Cisco was having trouble in their environment with changes to users and groups not being replicated after they were made immediately, and this could definitely be a source of problems. I am fairly sure that the issues you are having and others are specific to the Cisco environment as you said. Other than the logging configuration information that we have on the CSUM wiki and that Atlassian provides at http://confluence.atlassian.com/display/DOC/Requesting+External+User+Management+Support I don't really have much to offer in terms of additional info on logging configuration.

I think that at this point if you are still having trouble, you might want to work together with the Jira administrator and the DBAs (and any others) responsible for maintaining the other systems that Confluence uses directly or indirectly for user and group management. You could have them watch logs/queries, etc. and determine what is going on outside of Confluence. If you can determine that there is something that we are doing with the Jira integration part of CSUM that is wrong, I'd be more than happy to make any changes you suggest so that you can try it out, but unfortunately without that information from you and your team, I don't have the resources to help in that regard.

Best of luck, and please utilize Atlassian support if you can reproduce the issue in Confluence's administration area itself or via some direct Atlassian API into Confluence or Jira.

Thanks!
Gary
