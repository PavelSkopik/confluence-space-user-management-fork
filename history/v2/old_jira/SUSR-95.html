Gary S. Weaver added a comment - 11/May/09 9:27 AM

Steve,

Sorry I wasn't clear in last comment. Basically, in Confluence 2.8 a new way of managing resources (javascript, css) was introduced that was incompatible with the way used in various plugins prior to that version. Specifically this affected CSUM version 2.x up to and including 2.0.2. So I think that the plugin's javascript and css resources were not getting included in those versions of CSUM for Confluence 2.8.0 and up.

CSUM 2.0.3 included a fix for this such that at least as far as the plugin was concerned it would properly load its own javascript and css resources (and one of those is YUI) in Confluence 2.8 and higher. The big part of this fix was getting it to work so that it would be disabled and work just as it did before for Confluence version prior to 2.8.

However, apparently from the bug you found, there may some incompatibility with the way that we are loading those resources in CSUM 2.0.3 with Confluence 2.8 and higher, not in the plugin itself, but outside of the plugin in other pages. I'd like to have Atlassian look into it and provide suggestions, but I think that there is probably work to do in CSUM, so I'm leaning toward not bothering them with it at the moment.

I wish that I had time to work on it at the moment, but I don't. So I'll ask the others on the team if they can assist.

Thanks for your patience,

Gary

Ok, I'll go ahead and make that fix then. This is what Atlassian
recommends also:

from:
http://confluence.atlassian.com/display/DOC/Adding+Plugin+and+Module+Resources
—
Example of Resource Type: Stylesheet referring to Images

Stylesheets for your plugin may often refer to images also in your
plugin. In which case you would have to make both the stylesheet and
image(s) downloadable.

<resource type="download" name="my-images/"
location="com/example/plugin/myimages"/>
<resource type="download" name="my-style.css"
location="com/example/plugin/my-style.css"/>

Note: If you have multiple stylesheets and javascript resources defined,
you should put the resource defintions in a Web Resource Module.

To refer to your plugin images in a stylesheet, use a relative path
based on the resource name defined for the image (which is 'my-images'
in this case).
my-style.css

.my-class {
background-image: url(my-images/mypicture.gif);
}

To reference images already available in an application, you will need
to go up three parent directories like so:

.my-class {
background-image: url(../../../images/icons/confluence-logo.gif);
}
—
Thanks,

Gary
