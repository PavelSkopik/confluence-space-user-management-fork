Andy Brook added a comment - 09/Oct/07 9:13 AM

OK, it took a while but I finally figured it out. My 'what could go wrong' deploy process to the repository had a problem

I managed to deploy the poms as .pom.xml not .pom, which meant that the compile time dependencies would always be met when downloading fresh but the transitive dependencies would always be lost, and consequently never work live. I've verified this by clearing out the ldaputil tree in the local repo, and building csum from the command line, for 1.0.7, 1.0.8 and 1.0.9, they all now build. Gary, can you please check 1.0.9 works for you?

How it worked for me I can only assume I'm not poking it in the way you are.... When building, you should see the pom get downloaded, which was the hint, as 1.0.6 did , but the other did not...

I should have spotted this sooner :/

cheers!
