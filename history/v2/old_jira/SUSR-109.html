Gary S. Weaver added a comment - 31/Mar/10 7:58 AM

Dusty,

Since Atlassian changed their process several months ago, every time that a new release of Confluence comes out, unless all plugin managers update the supported versions of Confluence for that version of the plugin, the plugins cease to be available in Confluence.

This is an attempt to keep the plugins that are available in Confluence up-to-speed with Confluence changes, and it is a good idea (even though it bugs me as a plugin developer because I can't let things ride). However, since in theory nothing should have changed to the API with a minor version change, there is really no reason it shouldn't be compatible with 3.2 - as far as I know. Therefore, I just updated the supported versions to include 3.2, and you and everyone else should be able to pull it from Plugin manager now. I'm sorry.

If you have time, please continue to let us know if we slack off on stuff like this. It is a big help. It has been busy at work recently for me (and at home), and keeping the version updated is a ball I've obviously dropped.

However, I'll ask Jonathan/Atlassian and put in a request for plugins.atlassian.com that will send notifications to plugin developers in advance of a new Confluence release, or to see what list I need to get on to make that happen. I'll also put a note on the CSUM page asking for community help in testing 3.2 and future releases.

Thanks for your help,
Gary
