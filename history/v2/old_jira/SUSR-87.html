Peter,

Sorry to hear you are having trouble.

The CSUM plugin uses the Confluence API to create the users. If the users should not be created, that would be an issue with the Confluence UserAccessor (Confluence API) not behaving as it should. Please open up a support ticket with Atlassian for the version of Confluence you are using and let them know that the API should not allow addition of the username in question, supplying reasons to them why it shouldn't allow that, along with whatever system you are using in a lower layer for user management (if not Confluence) (like LDAP, etc.).

Before you send them that ticket, test to see whether a Confluence administrator in your confluence server can create a group with the same name via Confluence's own group management page (Administration (Upper Right) -> Manage Groups (lower left) -> create group). Whether it can or can't, tell Atliassian that in the support ticket. Assuming it is a bug, they would likely be more inclined to fix if it affects the Admin UI as well, although they would also obviously want their API to work correctly (since other plugins use it too).

In the meantime, you should also ask Atlassian to provide advice about how you can change or remove that group. You can likely just change the groupname via a DB update statement (assuming you are using Confluence) or similar for whatever means of user management you use. Obviously that is not optimal, which is why you should submit a support ticket with them about the API.

Have a good one,
Gary
