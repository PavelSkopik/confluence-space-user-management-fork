Jon,

Sorry to hear you are having trouble, but I hope I can help.

Here are some suggestions:

  • You might want to update to v2.0.3 of our plugin just for UI sake with Confluence v2.10 (v2.0.2 and previous versions of v2.x don't render well in Firefox from v2.8 on)
  • You might also want (but I can't say for sure) to upgrade your Confluence to the latest version if it isn't already (last I checked it was 2.10.3- they update quickly).
  • (Most importantly) The issue you are having seems to be with Confluence itself (as you said that you are trying to delete the group in the Confluence admin section after having deleted it in the csum plugin), so you'll want to file a support ticket with Atlassian. They will not officially support use of our plugin, but they do officially support the Confluence user-atlassian API that our plugin uses and that Confluence uses to manage users and groups. Basically it looks like the code in Confluence to handle group deletion is not handling the situation very well where a group has been deleted and someone else tries to delete it, if it is spitting it back to the user in the UI.
