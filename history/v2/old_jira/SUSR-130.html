Aditee,

Sorry to hear that. I know it may be a general "it doesn't work" bug, but please attached debug logging as specified in the Troubleshooting section in the wiki ( https://studio.plugins.atlassian.com/wiki/display/SUSR/Confluence+Space+User+Management+Plugin ):

  • Setup confluence user management debug logging using the config information here: http://confluence.atlassian.com/display/CONF34/Requesting+External+User+Management+Support
  • Add debug logging for the csum plugin by adding the following to that same logging config:

log4j.logger.csum=DEBUG, confluencelog

  • Ensure that you start capturing logging before you reproduce the issue and then copy the log immediately after you reproduce the issue. Starting to tail the log (tail -f (path to log) | ~/error.log) and stopping immediately may produce the most efficient log to debug the issue, unless the log rolls over while you are tailing. Ensure that it has captured debug logging. Preferably reproduce the issue in a test environment that is low traffic.
  • Be sure to scrub any private information from the logs, because when you attach it to a Jira issue, it will be public.
  • Create a JIRA issue in http://studio.plugins.atlassian.com/browse/SUSR (again, if you don't have a user account in that Jira, please add one for yourself) and include the error displayed in the UI and associated debug logs (see above) to that ticket.

Thanks,
Gary

Gary S. Weaver added a comment - 15/Mar/11 9:13 AM - edited

In addition, CSUM 2.0.6 works with Confluence 2.10.1, please use it if you can instead of CSUM 2.1 if you are using Confluence 2.x:
https://studio.plugins.atlassian.com/svn/SUSR/releases/permission-management-2.0.6.jar

It is unlikely that we will continue to support Confluence 2.x in CSUM 2.1.x+ unless it is relatively straightforward to fix or we get additional time or resources to help with development. Also, future versions of CSUM may only support Confluence 3.5+ and may cause removal of the "add user if in LDAP" option because of the switch to using embedded Crowd in Confluence 3.5+. I don't think that your environment uses the "add user if in LDAP" option, but you should probably work towards upgrading to Confluence 3.5 eventually, since the Jira integration the way it is done now will not be how it will work in Confluence 3.5+ from what I understand. CSUM does not currently support Confluence 3.5, but that is the current goal for future development. Please talk to Atlassian if you would like more information on Confluence 3.5+.

Thanks,
Gary
