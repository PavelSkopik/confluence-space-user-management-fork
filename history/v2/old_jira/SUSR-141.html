Gary S. Weaver added a comment - 20/Mar/12 8:55 AM

Dhana,

If I remember correctly we don't show read-only/LDAP groups anymore, since they cannot be managed. I'm not sure if that is what you are saying.

The local groups however should be showing up, if they are not LDAP, as writable non-LDAP groups whose name matches the space naming convention noted in the plugin configuration should in-theory still be able to be managed, and if they aren't, it is a bug.

Please provide debug logs (after you scrub (remove) information that you do not wish to share publicly) by attaching them to this ticket. Read the wiki page for the CSUM plugin to determine how to turn on debugging and troubleshoot. Only provide the debug log portion from the point immediately before the action you take in the wiki that is failing to show or do what you want it to show or do. A good way to do this is to tail the log and output to another file and then terminate the tail/output a few seconds after you reproduce. This allows me to focus on the issue vs. finding the issue.

I may not have time to get to this quickly in my free time and cannot guarantee an estimated time to fix. If you need a speedy resolution either:

  • Hire an Atlassian partner to donate their work and time to the free CSUM plugin to add support for LDAP. They are welcome to get some limited assistance from me with what time I can provide, although I can't promise to hold their hand through any work that needs to be done.
  • OR Contact me directly via email or provide me with an email, and we can try to work something out.

Note also (even though I don't think this pertains to your issue), if you need the previous functionality of CSUM to be able to look up users in LDAP separately from Confluence so that they can be added to local Confluence groups, that feature at the moment is no longer supported as part of the free volunteer support for the plugin. CSUM plugin's "LDAP username verification prior to local user add" functionality was only supported up until Confluence 3.5 when embedded Crowd was introduced. At that point an older version of the ldap library that Andy Brook had written years ago would no longer work. At first, it seemed that things were undocumented enough where I wasn't sure if we'd ever have time to work on it, but there is an example I found from Atlassian for how to use LDAP with embedded crowd in this authenticator:
https://bitbucket.org/jaysee00/example-confluence-sso-authenticator/src/381eb95ebc08/src/main/java/com/atlassian/confluence/seraph/example/ExampleSSOAuthenticator.java
However, I support the CSUM plugin in my free time, and I've not had much free time to reintegrate that "check LDAP indirectly to determine if a local user should be added" over the last several months.

Hope that helps,

Gary
