John Bowersox added a comment - 24/May/10 10:19 AM

Gary -

Thanks for the update and reassurance that the 3.x line will be supported as best you can moving forward.

I wanted to add some additional information on the problem we are having. We had some additional time last week to do some troubleshooting and found out why the 'GO' button is not working in our environment.

We have found that when clicking 'GO' for some reason the browser is adding the following to the new url - '52'

Here is an example from our test environment -

When you click on Next link on the bottom of the user grid, you go to this URL:
http://wctest.mars/customspacemgmt/permissions/docustompermissionsmanage.action?key=imcorpaffairs&selectedGroup=imcorpaffairs-ca%2Bglobal&pagerAction=nextPageUsers

When you use the form with the GO button, you go to this URL:
http://wctest.mars/customspacemgmt/permissions/docustompermissionsmanage.action?key=imcorpaffairs&selectedGroup=imcorpaffairs-ca%252Bglobal&pagerAction=skipToUser&recordNum=10&skipToUserBtn=Go

You can notice that for the selectedGroup parameter, the value is 'imcorpaffairs-ca%2Bglobal' in the first case and 'imcorpaffairs-ca%252Bglobal' in the second. The '52' are coming from nowhere.

When you obtain the error 'You haven't admin access', just go to URL bar and remove this '52' value and you have the correct page.

From what we can tell it is because the group name contains spaces. We run the same scenario on other groups without spaces and did not run into any issues.

Gary S. Weaver added a comment - 24/May/10 2:07 PM

John,

That looks like a URL escaping issue to me. 25 is hex for the % char: http://www.blooberry.com/indexdot/html/topics/urlencoding.htm

The velocity code in src/main/resources/templates/permissionmgmt/group-pager-actions.vm is:

#if($groups.total > $groups.countOnEachPage)
<form action="docustompermissionsmanage.action" class="pageSkip">
	<fieldset>
	<input type="hidden" name="key" value="$generalUtil.doubleUrlEncode($key)"/>
	<input type="hidden" name="selectedGroup" value="$generalUtil.doubleUrlEncode($action.selectedGroup)"/>
	<input type="hidden" name="pagerAction" value="skipToGroup"/>
	<label>$action.getText("csum.skip.byNum"):</label>
	<input type="text" name="recordNum"/>
	<input type="submit" class="submitBttn" id="skipToGrpBtn" name="skipToGrpBtn" value="Go"/>
	</fieldset>
</form>
#end

So basically, it looks like in a number of places I (probably it was me at least) got too happy with escaping in attempt to be more secure and didn't test enough with wierd names- OR something has changed in Confluence that I just missed or there is some bug with that generalUtil. In any case, I've attached a build that doesn't use that in any value="..." and let me know if it works for you.

Thanks!

Gary
