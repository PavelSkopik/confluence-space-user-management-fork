Gary S. Weaver added a comment - 27/May/09 9:20 AM

Vijay,

A few questions for you:

  • If you browse memberships of that same group in Confluence Administration area, is it similarly slow? The CSUM plugin uses the same part of the API that Confluence itself uses for user and group browsing that is built upon a Confluence API "Pager" that should in theory not load up every user for the default Confluence user management (DB) implementation, and if whatever you are using for user/group management in Confluence is slow, that is a support issue you'll want to take up with Atlassian.
  • What are you using for user management? If you are using built-in Confluence user management, be sure that you are using atlassian-user schema vs. OS User schema which is very slow and not scalable. This is a problem with the OS User implementation (not our plugin) and Atlassian can provide support around pointing you to the correct documentation if needed.

I would definitely focus on trying to reproduce this in the Confluence administration area (not in the plugin) so you can get Atlassian support to assist you. However, if you find that there are ways that we are calling the Confluence API that we should be doing differently (which is not likely for Confluence 2.6.2 I think) then please let us know.

Good luck,

Gary
