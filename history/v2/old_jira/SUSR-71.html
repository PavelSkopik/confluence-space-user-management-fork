Description

Hi !

We have a strange case of superfuous LDAP requests when using Confluence + CSUM + LDAP integration.

The full story:

A Space admin wants to add a LDAP User to a Confluence Local Group with the help of CSUM plugin v2.0rc3:

  • The guy selects the group where the user is going to be added
  • Then he selects the right user with the help of a search engine and enters a query looking like « username », « starts-with », « <LoginUser> ».

That is triggering 3 LDAP search queries:

Filter " (&(objectClass=extendedPerson)(uid=<LoginUser>*))"
Filter " (&(objectClass=extendedPerson)(uid=<LoginUser>))"
Filter " (&(objectClass=extendedPerson)(uid=<LoginUser>*))"

Do you have an idea why the same request is sent 3 times ?
Is it something happening on the CSUM side or within Confluence ?

Thanks,
Ludovic

I too have noticed this for a long time, but made the assumption that every macro and bit of content that was loaded needed to be authorized for the current user, sometimes I get 10's of such things. I also make the assumption that there is some cacheing going on and that this loggin is occurring at a lower level, not necessarily requiring LDAP queries EVERY time.

I would suggest opening a ticket with Atlassian to query the level of authorizations that are going on (authentication should only happen once due to the seraph cookie being in place).

happy to be involved in this, not sure there is an issue though, we use AD and CSUM so can assist in debug, though my time is being diverted from Confluence to other stuff...
