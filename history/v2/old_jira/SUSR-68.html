Description

I seem to be having problems with the plug-in when I am using Crowd. First off, when I go to configure the plug-in, I have to choose Confluence for the user management even though it is really Crowd. Under Auto-provision users using LDAP info, if I choose yes (which I want to), it wants to look in either the osuser.xml or atlassian-user.xml for LDAP info, I cannot just put in the LDAP info. Using Crowd, I do not user osuser.xml and I do not use the LDAP section of Atlassian-user (it is commented out). With that said, I check "No" on this option to complete the configuration. When I go to create/add a group to a space, it allows me to create the group but will not add any users - I actually get an error on the page when I try to add users. The error is on line 230 - object doesn't support this property or method.

Gary S. Weaver added a comment - 21/Nov/07 8:03 AM

Laurie,

Justen from Atlassian responded about this problem in comments of page: http://confluence.atlassian.com/display/CROWD/SOAP+API to my question about this particular issue:

Justen Stepka - Nov 20, 2007 20:37 - "Through the plugins API you can use the user API as usual which will attempt to push changes through Crowd to whatever directory store is being used."

When Confluence is integrated with Crowd, the Custom Space User Management plugin (which uses UserAccessor in the Confluence API) should be able to add/remove groups, add/remove group memberships, and add/remove users if those same functions would work in Crowd itself (i.e. as long as Crowd has rights to do those things and Confluence is correctly configured to use Crowd and there are no bugs/incompatibilities between the versions of Confluence and Crowd that you are using).

This means that the Custom Space User Management plugin should in theory support Crowd integration with Confluence.

I'm assuming that it is either the case that Crowd itself is not able to add group memberships into Active Directory in your case because of some configuration issue with Active Directory, Crowd, and/or Confluence, or this is a bug in some version of those that you are using that you'd need to take up with Atlassian.

If you want to get the C.S.U.M. plugin working with Confluence and Crowd at this point, the way I'd suggest doing it is to do the following:

  • In Confluence, go to Administration -> Manage Groups and try to add a user to a group
  • From what Justen said, and from looking at your logs, I'm guessing that this will fail
  • Open up a support ticket with support.atlassian.com and attach your logs to the ticket, and tell them specifically what you did to produce that problem (and try it again so you can say that the problem is reproducible)
  • You shouldn't mention the C.S.U.M. plugin because it has nothing to do with the problem. If you do mention it, the support person will likely say that it is a 3rd-party plugin and they only support plugins that Atlassian has written, so that's why I would say don't mention it.
  • Whether it is a configuration issue or a bug, Atlassian should be able to help you with this problem

If that is not the case, and this issue only affects the C.S.U.M. plugin (for reasons I can't imagine), then let us know. Either way, please comment back on this page with whether you were able to get it to work or not. We'd love to have a user of the plugin that got it to work successfully with Crowd. I know others will likely be planning to use it with Crowd soon, so it'd be great for them to know how your issues were resolved.

Thanks and happy Thanksgiving! (if you are in the states)

Gary

P.S.- this ticket will be closed unless we get a response
